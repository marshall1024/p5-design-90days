### ✅ 一、基本设计原则

#### 📚 学习资料推荐：

| 名称                                   | 类型 | 简介                                        |
| -------------------------------------- | ---- | ------------------------------------------- |
| 《Clean Architecture》Robert C. Martin | 书籍 | 阐述架构设计的基本原则，如SOLID、依赖反转等 |
| 《架构整洁之道》                       | 书籍 | 中文译本，适合深入理解架构与模块边界        |
| SOLID 原则官方文档 / 博客              | 文章 | 搜索关键词：SOLID principles Java/C#/Python |

#### 🎯 学习方法：

1. **先理解，再背诵**：不要死记硬背，先通过小项目实践理解每个原则。
2. **代码审查训练**：找一段开源代码，尝试指出是否违反SOLID原则。
3. **写博客总结**：每学一个原则，用自己的话写一篇文章，强化理解。
### ✅ 二、常见设计模式
#### 📚 学习资料推荐：

| 名称                                      | 类型 | 简介                             |
| ----------------------------------------- | ---- | -------------------------------- |
| 《设计模式：可复用面向对象软件的基础》GoF | 书籍 | 23种经典设计模式，必读           |
| 《Head First 设计模式》                   | 书籍 | 图解+幽默，适合初学者入门        |
| Refactoring.Guru 网站                     | 网站 | 图解设计模式，支持中文，强烈推荐 |
| 百度/阿里技术博客                         | 文章 | 搜索关键词：设计模式实际应用案例 |

#### 🎯 学习方法：
项目驱动学习：每学一个模式，尝试在项目中用上（如用工厂模式重构对象创建）。
模式对比法：对比相似模式（如策略 vs 状态），写表格总结差异。
刷题巩固：在 LeetCode 或设计题中识别模式应用场景。

### ✅ 三、设计工具使用
#### 📚 学习资料推荐：

| 工具                                            | 学习资源                                      |
| ----------------------------------------------- | --------------------------------------------- |
| PlantUML                                        | 官方文档 + B站教程（搜索：PlantUML 快速入门） |
| draw\.io / diagrams.net                         | 在线工具，适合画架构图、流程图                |
| Enterprise Architect / StarUML                  | 专业建模工具，适合大型项目                    |
| IntelliJ IDEA + 插件（如 PlantUML integration） | 高效画图+代码联动                             |

#### 🎯 学习方法：
从模仿开始：找一张优秀架构图，尝试用工具自己画一遍。
建立图库模板：常用图（时序图、类图、部署图）做成模板，复用高效。
输出驱动：每做一个设计，强制输出一张图，哪怕很简单。

### ✅ 四、项目设计流程

#### 📚 学习资料推荐：
| 名称                             | 类型   | 简介                                  |
| :------------------------------- | :----- | :------------------------------------ |
| 《软件架构设计：程序员进阶之路》 | 书籍   | 从需求到架构的完整流程                |
| 阿里技术公众号文章               | 文章   | 搜索关键词：阿里架构设计流程、DDD实战 |
| 极客时间《架构实战案例解析》     | 视频课 | 真实项目架构演进案例讲解              |

#### 🎯 学习方法：

1. **模拟一个项目**：从0设计一个「在线商城」或「博客系统」，走完完整流程。

2. **角色扮演法**：一个人扮演产品、架构、开发，模拟需求评审、技术方案设计。

3. **写设计文档**：强制输出《技术方案文档》《架构评审报告》，训练结构化表达。

### ✅ 五、数据库设计

#### 📚 学习资料推荐：

| 名称               | 类型 | 简介                                         |
| ------------------ | ---- | -------------------------------------------- |
| 《高性能MySQL》    | 书籍 | 数据库设计、索引、优化必读                   |
| 《数据库系统概念》 | 书籍 | 理论扎实，适合打基础                         |
| 阿里/美团技术博客  | 文章 | 搜索关键词：分库分表、索引优化、范式与反范式 |
| dbdiagram.io       | 工具 | 在线画ER图，支持导出SQL                      |

#### 🎯 学习方法：

1. **从业务出发设计表**：不要一上来就画ER图，先写用例，再反推表结构。
2. **反范式训练**：尝试设计一个电商订单系统，思考何时反范式、如何分表。
3. **性能压测验证**：用JMeter或自写脚本压测接口，观察索引是否生效。

### 90 天冲刺计划

1. 能输出一份《xx 系统技术方案》文档（含业务描述、用例图、模块图、时序图、表设计、接口定义）。
2. 能画标准的 C4 + UML 图（PlantUML 代码级）。
3. 代码层面能落地：SpringBoot + MyBatis-Plus + 分模块 + 设计模式 + 单元测试。
4. 面试可以讲清楚“为什么这样设计、为什么这样分库、为什么用策略模式”。

阶段 1  打地基（第 1-30 天）
目标：把“设计原则 + 常用模式”变成肌肉记忆；能看懂/改画 UML。

| 时间        | 任务                                                         | 资料 & 工具                                                  | 验收标准                                                     |
| :---------- | :----------------------------------------------------------- | :----------------------------------------------------------- | :----------------------------------------------------------- |
| 第 1-7 天   | ① 通读 SOLID 5 原则，② 各写 1 个 Java 例子 push GitHub       | Refactoring.Guru 中文 + 《Clean Architecture》第 1-8 章      | 5 个单元测试，每个原则 1 个 case，README 用 100 字解释       |
| 第 8-14 天  | 掌握 10 个高频模式：单例、工厂、策略、模板、建造者、代理、装饰器、观察者、责任链、适配器 | 极客时间《设计模式之美》前 20 集 + GoF 原书对应章节          | 每个模式：① 画 1 张 UML 类图（PlantUML 代码）② 写 1 个 SpringBoot 小 demo（可合并到同一仓库） |
| 第 15-21 天 | 学会画 4+1 视图：用例、逻辑、实现、部署、进程                | B 站搜 “C4 model 15 分钟” + PlantUML 官方模板                | 把“用户下单”场景画出：① 用例图 ② 逻辑视图（模块图） ③ 时序图 |
| 第 22-30 天 | 小项目实战：给上面 10 个模式统一做一个“营销发券”骨架工程     | 需求：平台发券、用户领券、订单核销。必须用到策略+工厂+模板+观察者 | ① 能跑通本地 Swagger ② README 贴架构图 ③ 写 3 条 JUnit 覆盖主流程 |
